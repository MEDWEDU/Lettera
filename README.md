Проект уже содержит реализованный пользовательский интерфейс, расположенный в папке client. Он разработан на React (с использованием TypeScript), и переход на Next.js не требуется — мы продолжаем работать исключительно на React.

Обходимся без технологий монорепозитория - оставляем просто все приложения в одном репозитории, но при этом они должны запускаться по отдельности друг без друга - либо проводим вариант разных репозиториев для каждого проекта crm-client, server и client

Однако само приложение на данный момент требует доработки: необходимо реализовать недостающую функциональность и интегрировать полноценный бэкенд-сервис (API).

Все ключевые материалы проекта уже подготовлены и доступны в следующих файлах:

Техническое задание: ТЕХНИЧЕСКОЕ ЗАДАНИЕ: LETTERA.md

Описание экранов и пользовательских сценариев: ТЕКСТОВЫЙ ПРОТОТИП ДИЗАЙНА: LETTERA.md

Архитектура базы данных: АРХИТЕКТУРА БАЗЫ ДАННЫХ: LETTERA.md

Scrum-доска задач: Lettera — SCRUM Task Board.md — её необходимо поддерживать в актуальном состоянии в процессе разработки.

План реализации:

Frontend (client)
Довести до завершения в соответствии с ТЗ и текстовым прототипом.
Использовать существующий дизайн и стилевые решения — от Next.js отказались окончательно.

Backend (server)
Полностью реализовать API, описанное в техническом задании и согласованное с архитектурой БД.

CRM-панель (crm-client)
Разработать отдельное админское приложение на React + TypeScript + Vite.

Дизайн CRM может быть предложен с помощью AI-инструментов, но должен быть визуально и стилево согласован с основным приложением (client) — использовать те же цвета (включая акцентный #2290FF), компоненты и общий подход к UI.

Разработка ведётся строго в соответствии с утверждённым ТЗ, с учётом всех описанных требований и ограничений.
